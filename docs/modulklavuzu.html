Türkçe Entegrasyon Modülü Yazma Kılavuzu

Tedarikci Entegrasyonu Modül Yazma Klavuzu Prowered by http://www.duzgun.com/

T.E entegrasyonu yazılımı güçlü çekirdek altyapısı ile açık kaynak yönetim imkanını kullanıcılarına sunmaktadır. Bu nedenle tedarikçi modülleri açık kaynak koddur.

Tedarikçi Entegrasyonunun çekirdeğine T.E. Motoru denilmektedir. T.E modülleri, açık kaynak kod olduğu için herhangi bir yere bağımlılığınız yoktur. Herhangi bir php yazılımcısı bu klavuz eşliğinde değişen XML şekilllerini ve data formatını görüp kullandığı modülü düzenleyebilir. Yine php yazılımcısı olmayanlar herhangi bir php yazılımcısına güncel modülü yazdırabilir.

XML yapısı değişikliğinden kaynaklı olmak üzere modül güncelleme işlemini kullanıcı kendi imkanları ile yapabilir. Kişi modülleri istediği gibi geliştirebilir ve yazabilir. Bunun için bir engelleyici kuralda yoktur.

T.E kullanıcılarının Modül konusunda sıkıntı çekmemeleri ve bağımlılıklarını ortadan kaldırmak için T.E modül yazma klavuzu yayınlanmıştır.

Tedarikçi Entegrasyonu Modül Yapısı (Modül yazmak ve güncellemek için Php diline hakim olmanız yada php ye hakim olan birine yazdırmanız gereklidir.)

Tedarikçi Entegrasyonu modülleri tekbir php dosyasından oluşmaktadır.

http://www.xmladresim/dosyam.xml ile yazılan data ulaşım satırı size özeldir ve tedarikçiniz ile sizin arasındaki yasal sözleşmelere bağlıdır. Bu adresi her modüle siz yazarsınız. Bu adresi paylaşmamalısınız. Tedarikçi Entegrasyonu modülünde yazdığınız yukarıdaki php kodu sayesinde size ozel olan bu dosya sitenize güvenli olarak indirilir. Fonksiyonu T.E arayüzünde xml oku yönteminde Ürünler butonuna tıklandığından çalıştırılan fonksiyondur.

Bu fonksiyon ile ilk modülümüzü yazalım.
XML örneği aşağıdaki gibi olsun.

yukarıdaki taglar arasına kendiniz birşeyler yazabilirsiniz. Php dilindeki simplexml yöntemi ile xml okuma işlemini kolaylıkla yapabilmektesiniz. Öncelikle xml lerde bulunan tag(etiket)lar herhangi bir değişkene atanmalı. Bu şekilde kullanımı çok daha basit olmaktadır.

Modülün kodlamasına devam ederken $this→xml değişkeniyle indirilmesi istediğimiz xml in adını yazarak simplexml sınıfına aktarıyoruz. simplexml sınıfının bir kopyası simplexml_load_file fonksiyonu ile $xmlData değişkenine aktarılmaktadır. $xmlData→{$baslangicetiketi} değişkeni ile XML yapısındaki urun etiketine erişebilmekteyiz. urun etiketi array(dizi) özellikli olduğu için foreach ile herbir urun etiket içeriği $product değişkenine atanarak saydırılmaktadır.

<?php
$xmlData = simplexml_load_file($xml_directory.$this->code."/urunler.xml");
foreach($xmlData->{$baslangicetiketi} as $product)
{
}
?> 

foreach döngüsünden ürün bilgilerine etiket isimlerini yazarak ulaşabilmekteyiz. Bu ulaşım xml data şekli ve simplexml in bize sunduğu yöntemlerle direk bağlantılıdır.
foreach döngüsü içine aşağıdaki kodu ekleyelim.

<?php
$this->category = array('code' => $product->{$kategori_kodu},
                                 'cname' => $product->{$kategori_adi},
                                 'parentcode' => '');
core_category();
?>

$this→category değişkeni ve core_category() fonksiyonu özel isimlidir.
$this→category değişkenine hash formatlı datalar
code = kategori belirteci
cname = kategori görünür adı
parentcode = kategori belirtecine bağlı olup olmadığı(yazılan kategorinin altında oluşturulur boş ise ana kategori demektir.)
anlamına gelir.

core_category(); fonksiyonu T.E motorundaki bir fonksiyondur ve $this→category değişkenindeki bilgileri T.E motoruna göndererek gönderilen bilgilerin işlenmesini sağlar. Bu durumda kategori henüz aktarılmamışsa T.E kontrol tablolarında oluşturulur yada güncellenirler.

Bu işlemlerle 1. bölümde limit ve start çevre değişkenleri ile sıradaki okunacak kaydın hangi kayıttan başlayacağını ve ne kadar kayıt okuyacağını belirtir.

2. bölümde ise okunacak kayıt mikatından fazla kayıt varsa işlem okuma işlemi yapılmadan bir sonraki parçalışı işlem devamı için döngü boş geçilir.

3. İşlem tamamlanmışsa return null değerini otomatik döndürmüş olur eğer return 1 ise indirme işlemi parçalı yapılmıştır ve devam etmektedir anlamına gelir.

Ürün disable / enable senkronizasyon özelliğinin Modül eklenmesi.

config.php dosyasında aşağıdaki kod olacaktır.

<?php
/*ürün disable / enable senkronizasyon özelliği tedarikçi modülüne bağımlı olsun ise true olmalıdır. 
Modülde isdeleted alanı ile düzenleme gerektirir. 
true kullanıldı ise e-ticaret sisteminizden ürünü disable ettiğinizde 
isdeleted modülde tanımlı değilse güncellemelerde otomatik enable e dönüşecektir. 
Bu nedenle modülde isdeleted kullanılmadığı durumda false tercih edilmelidir.*/
$PUPDATESTATUS = true;
?> 

açıklama satırından da anlaşılacağı gibi modülde isdeleted özelliği kullanılabilmektedir. modülde isdeleted özelliğinin kullanılabilmesi için config.php deki $PUPDATESTATUS = true; değişkeninin değerinin true olması gerekir. Bu değer false ise isdeleted kullanımının etkisi olmaz.

isdeleted özelliği modülde $this→product hash değişkenine eklenir. isdeleted index inin aldığı değer 1 ise ürün e-ticaret sistemine durumu disable olarak eklenir yada güncellenir. 0 ise enable olarak eklenir.

E-ticaret yönetim panelinizden ürün durumunu disable yapsanızda isdeleted ile enable geliyorsa durumu otomatik enabled e dönüşecektir.

disabled = pasif
enabled = aktif olma durumudur.

isdeleted kullanım örneği aşağıdadır.

<?php
$this->product = array('pcode' => $product->{$urun_kodu},
                                'pname' => $product->{$urun_adi},
                                'price1' => $product->{$urun_fiyati},
                                'currency' => $product->{$parabirimi},
                                'tax' => 'KDV DAHIL',
                                'stock' => $product->{$urun_stogu},
                                'catid' => $product->{$kategori_kodu},
                                'brand' => 'My Brand',
                                'desc' => $product->{$urun_aciklamasi},
                                'isdeleted' => ((empty($product->{$urun_resmi}))?1:0)
);
?> 

örnekteki uygulamayla eğer ürün resmi yoksa ürünün pasif olarak aktarılacaktır.
